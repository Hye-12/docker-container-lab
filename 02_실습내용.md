# 수행 내용

## 1. Docker Repository 등록

DNF 플러그인을 설치한 후 Docker 공식 Repository를 등록하였다.

```bash
sudo dnf -y install dnf-plugins-core
sudo dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
````

## 2. Repository 오류 해결

Repository 등록 과정에서 URL 끝에 '.'(점)이 포함되어 잘못된 Repository 파일이 생성되었다.

생성된 파일:

```text
download.docker.com_linux_centos_docker-ce.repo.repo
```

잘못 생성된 파일을 삭제하고 캐시를 재생성하였다.

```bash
sudo rm -f /etc/yum.repos.d/download.docker.com_linux_centos_docker-ce.repo.repo

sudo dnf clean all
sudo dnf makecache
```

## 3. Docker 설치

Docker Engine과 관련 패키지를 설치하였다.

```bash
sudo dnf install -y docker-ce docker-ce-cli containerd.io
```

## 4. Docker 서비스 시작

```bash
sudo systemctl enable --now docker
```

## 5. 설치 확인

Docker가 정상적으로 동작하는지 확인하기 위해 hello-world 이미지를 실행하였다.

```bash
docker run hello-world
```

정상적으로 실행되어 Docker Engine과 Docker Hub 통신이 가능함을 확인하였다.

## 6. 이미지 및 컨테이너 관리

이미지 확인

```bash
docker images
```

컨테이너 확인

```bash
docker ps -a
```

컨테이너 삭제

```bash
docker rm <컨테이너ID>
```

이미지 삭제

```bash
docker rmi <이미지명>
```

## 7. Apache 이미지 다운로드

```bash
docker pull httpd
```

이미지 확인

```bash
docker images
```

## 8. Apache 컨테이너 생성

컨테이너 4개를 생성하여 서로 다른 포트로 서비스를 제공하도록 구성하였다.
** 도커는 생성시 동일한 이름을 허용하지 않는

```bash
docker run -itd -p 60080:80 --name h1 httpd
docker run -itd -p 60180:80 --name h2 httpd
docker run -itd -p 60280:80 --name h3 httpd
docker run -itd -p 60380:80 --name h4 httpd
```

실행 상태 확인

```bash
docker ps
```

포트 매핑 현황

| 컨테이너 | 호스트 포트 | 컨테이너 포트 |
| ---- | ------ | ------- |
| h1   | 60080  | 80      |
| h2   | 60180  | 80      |
| h3   | 60280  | 80      |
| h4   | 60380  | 80      |

## 9. 컨테이너 정리

실행 중인 모든 컨테이너를 강제로 종료 및 삭제하였다.

```bash
docker rm -f $(docker ps -aq)
```

## 11. Nginx 컨테이너 생성
